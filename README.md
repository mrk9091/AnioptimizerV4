# ⚡ AniOptimizer V4

> **Native C Gaming Daemon for Android — Magisk / KernelSU Module**

AniOptimizer is a root module that runs a native C daemon (`anioptimizerd`) in the background to intelligently optimize your Android device for gaming. It goes far beyond simple governor tweaks — it watches your games in real time, learns from your device, and applies fine-grained kernel and scheduler controls to deliver smoother frames with less heat.

---

## ✨ Features

### 🧠 Adaptive Learning Engine
The daemon builds a behavioral fingerprint of each game over time. It learns your device's thermal curve, frame patterns, and workload spikes — and uses that knowledge to pre-tune settings before problems arise, not after.

### 🌡️ Predictive Thermal Management
AniOptimizer uses **EMA-velocity + acceleration** prediction to detect thermal runaway before it happens. It fires a pre-cooling sequence **5–10 seconds before** your device hits the hard temperature limit, preventing throttle-induced frame drops.

- Hard limit: 75°C → drops to SAFE state immediately  
- Critical limit: 80°C → SAFE state + error log  
- Configurable via `default.conf`

### 🎮 Per-Game Profiles
Every game in the supported list has a hand-tuned profile with:
- **Thread niceness** (`nice`) for render/main thread priority
- **Target FPS** hint passed to ADPF (Android Dynamic Performance Framework)
- **Force Performance** lock for GPU-heavy titles (e.g. Genshin Impact, Wuthering Waves)
- **Touch boost suppression** for turn-based / strategy games where it wastes power

### 📲 Touch Boost
Detects touch input and instantly bumps the display polling rate to 120Hz / 240Hz for the duration of the interaction. Boost duration is configurable (`touch_boost_ms`, default 100ms).

### 🔋 Charging Current Limiter
During **Performance** and **Turbo** states, the charging current is capped (default: 70% of max) to reduce heat generated by simultaneous gaming + charging. Configurable or disableable in `default.conf`.

### 🔄 Adaptive Poll Intervals
The daemon dynamically adjusts its own polling rate based on the current performance state, so it uses minimal CPU when the device is in menus or low-load situations:

| State | Interval |
|---|---|
| Turbo | 100 ms |
| Performance | 200 ms |
| Balanced | 500 ms |
| Low / Safe | 1000–2000 ms |
| Menu / Lobby | 2000 ms |

### 🧵 Background App Isolation
Non-game background processes are deprioritized when a supported game is detected, freeing CPU bandwidth and reducing memory pressure for the foreground game.

### 📉 Stutter Detection
The daemon monitors frame delivery patterns and flags stutter events internally. When sustained stutter is detected, it escalates the performance state or adjusts thread priorities to recover smoothness.

### ⚙️ uClamp Integration
Uses Linux `uclamp` (CPU utilization clamping) to set minimum and maximum CPU utilization bounds on game threads, giving finer control than governor alone.

### 🖼️ SurfaceFlinger / FPSGO Tuning
Interfaces with SurfaceFlinger and (where available) MediaTek FPSGO to align the display pipeline and frame scheduler with your game's target FPS.

### 🔀 IRQ Affinity
Pins interrupt handlers to specific CPU cores to reduce latency and prevent ISR jitter from interfering with render threads.

### 🔥 PSI Thermal Awareness
Monitors Linux **Pressure Stall Information (PSI)** metrics to detect CPU/memory pressure and react before the system hits hard thermal walls.

### 🛠️ `anictl` CLI
A companion command-line tool for use in Termux after reboot:

```sh
anictl status    # Show daemon state, active game, current profile
anictl reload    # Hot-reload config without rebooting
anictl boost     # Manually trigger a Turbo boost
```

Config can also be reloaded by sending `SIGHUP` to the daemon:
```sh
kill -HUP $(cat /data/adb/anioptimizer/daemon.pid)
```

| Category | Games |
|---|---|
| FPS / Shooter | Call of Duty Mobile, Crossfire, Warzone Mobile |

## ⚙️ Configuration

All config lives in `/data/adb/anioptimizer/` after install. Edit files there — changes in the module directory are ignored after first boot.

### `default.conf` — Key Options

| Key | Default | Description |
|---|---|---|
| `target_package` | `auto` | Package to optimize, or `auto` for detection |
| `hard_temp_limit` | `75000` | Hard thermal ceiling in millidegrees (75°C) |
| `critical_temp_limit` | `80000` | Critical thermal ceiling (80°C) |
| `predictive_thermal` | `true` | Enable EMA predictive pre-cooling |
| `touch_boost_enabled` | `true` | Enable touch-triggered polling boost |
| `touch_boost_ms` | `100` | Duration of touch boost in ms |
| `limit_charging` | `true` | Cap charging current during performance states |
| `charging_limit_pct` | `70` | % of max charge current allowed |
| `debug_log` | `false` | Enable verbose debug logging |

### `profiles.conf` — Per-Game Keys

```
com.example.game=nice:-18,target_fps:90,force_performance:true,no_touch_boost:false
```

| Key | Description |
|---|---|
| `nice` | Thread niceness for render thread (-20 to 19) |
| `target_fps` | FPS target for ADPF hint (0 = auto-detect) |
| `force_performance` | Lock minimum state to PERFORMANCE |
| `no_touch_boost` | Disable touch boost for this game |

## 🚀 Installation

1. Flash `AniOptimizerV4.zip` via **Magisk** or **KernelSU**.
2. Reboot your device.
3. Open **Termux** and run `anictl status` to verify the daemon is running.

> **Note:** The module requires the pre-compiled `anioptimizerd` binary. Build it with `termux_build.sh` if not included.

| File | Contents |

Max log size is 512 KB by default (`log_max_bytes=524288`), then rotated.
